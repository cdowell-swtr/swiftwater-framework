    CLI reference

    app

    Framework CLI — scaffold solid, observable, testable Python projects.

    Usage:

     [OPTIONS] COMMAND [ARGS]...

    Options:

      --install-completion  Install completion for the current shell.
  --show-completion     Show completion for the current shell, to copy it or
                        customize the installation.

    audit

    Run all review agents in-process and write audit-report.md + meta.json.

    Usage:

     audit [OPTIONS]

    Options:

      --target TEXT       'framework' or 'project' (default: auto-detect).
  --agent TEXT        Restrict to this agent. Repeat for multiple.
  --out-dir TEXT      Output directory.  [default: .framework/audit/latest]
  --backend TEXT      Backend: 'api' or 'subagent' (default: from config).
  --snapshot          Force every agent into snapshot mode.
  --since TEXT        Force delta mode against a git ref/SHA or baseline dir.
  --resume            Resume from a prior checkpoint.
  --fresh             Discard a stale checkpoint and restart.
  --preserve-as TEXT  After finalize, copy the audit tree into this dated
                      baseline dir.
  --force             Required to overwrite a non-empty --preserve-as target.

    check

    Report whether a newer framework release is available.

    Usage:

     check [OPTIONS]

    dev-combos

    Emit the render-matrix battery combinations as JSON (framework dogfooding CI).

    Usage:

     dev-combos [OPTIONS]

    Options:

      --strategy TEXT        representative | pairwise | sample | broad  [default:
                         representative]
  --seed INTEGER         Seed for the random rotation.  [default: 0]
  --sample-size INTEGER  Random combos added to the broad/sample set.
                         [default: 6]

    downskill

    Remove a battery from a project (deletes its files; preserves migrations), then run its tests.

    Usage:

     downskill [OPTIONS] NAME BATTERY

    Options:

      NAME     Path to the project.  [required]
  BATTERY  Battery to remove, e.g. 'webhooks'.  [required]
  --force  Remove even if the battery appears in use.

    eval

    Run golden fixtures through the review agents and score recall/precision (spec §20).

    A malformed agent response (FindingsParseError) is scored as no findings for that single repeat and reported via a WARNING (and a parse_error marker in --findings-out); only anthropic.APIError aborts the whole run.

    Usage:

     eval [OPTIONS] [AGENT]

    Options:

      [AGENT]              Evaluate only this agent (default: all registered).
  --fixtures TEXT      Fixtures root directory.  [default:
                       tests/eval/fixtures]
  --repeat INTEGER     Runs per fixture; rates are averaged.  [default: 1]
  --require-fixtures   Fail if an evaluated agent has no fixtures.
  --require-key        Fail (not skip) if ANTHROPIC_EVAL_API_KEY is unset.
  --findings-out TEXT  Directory to write per-(agent,fixture,repeat) findings
                       JSON for diagnosis.
  --backend TEXT       Model backend: 'api' (paid) or 'subagent' (free claude
                       -p). Default None: resolves from
                       FRAMEWORK_REVIEW_BACKEND env or review.toml config.

    eval-analyze

    Produce scorecard + cost + recall/fp diagnosis + threshold proposal from per-call eval records (the artifacts written by framework eval --findings-out).

    Usage:

     eval-analyze [OPTIONS] FINDINGS_DIR

    Options:

      FINDINGS_DIR          Directory written by `framework eval --findings-out`.
                        [required]
  --out TEXT            Write the report to this path (else print to stdout).
  --thresholds TEXT     Current thresholds file (used as the comparison
                        baseline).  [default:
                        tests/eval/fixtures/thresholds.yaml]
  --margin FLOAT        Margin applied when proposing thresholds (recall_min =
                        recall - margin).  [default: 0.1]
  --strict              Exit code 2 if any drift is detected (used by the gate
                        context).
  --scorecard-dir TEXT  Write the tune artifact set (scorecard.md,
                        thresholds.proposal.yaml, apply.md, meta.json) into
                        this directory.

    gate

    Run the in-process review gate on the staged set and write marker.json.

    Skip-neutral: if no AI review backend is enabled the gate writes a PASS marker and exits 0 — a backend-less commit is never blocked.

    Usage:

     gate [OPTIONS]

    Options:

      --backend TEXT  Backend: 'api' or 'subagent'.
  --out-dir TEXT  Output directory for findings.  [default:
                  .framework/audit/latest]

    integrity

    Verify the framework scaffolding in the current project is intact.

    Usage:

     integrity [OPTIONS]

    Options:

      --ci                CI mode: skip gitignored existence checks (fresh
                      checkouts).
  --allow-drift TEXT  Record a managed file as intentionally diverged.

    new

    Scaffold a new project from the framework template.

    Usage:

     new [OPTIONS] NAME

    Options:

      NAME                   Human-readable project name  [required]
  --python-version TEXT  Python version to target  [default: 3.12]
  --with TEXT            Activate a battery (repeatable), e.g. --with
                         websockets.
  --alerts TEXT          Alert channels, comma-separated:
                         webhook,slack,email,pagerduty.

    restore

    Re-fetch a canonical framework file, discarding local edits to it.

    Usage:

     restore [OPTIONS] FILE

    Options:

      FILE    Path (relative to the project root) of the framework file to
          restore.  [required]

    review

    Run a Layer-3 review agent over the PR diff and post a GitHub Check Run.

    Usage:

     review [OPTIONS] AGENT

    Options:

      AGENT                Review agent name, e.g. 'security'.  [required]
  --findings-out TEXT  Write this agent's findings JSON to this path (for
                       aggregation).
  --target TEXT        Review target: 'project' (default) or 'framework'.
                       [default: project]
  --backend TEXT       Model backend: 'api' (paid) or 'subagent' (free claude
                       -p). Default None: resolves from
                       FRAMEWORK_REVIEW_BACKEND env or review.toml config.

    review-agents

    Print the JSON array of review agents active for the event (drives the CI matrix).

    Usage:

     review-agents [OPTIONS]

    Options:

      --event TEXT   GitHub event name (default: $GITHUB_EVENT_NAME).
  --target TEXT  Review target: 'project' (default) or 'framework'.  [default:
                 project]

    review-aggregate

    Aggregate per-agent review findings into one sticky PR comment (prints on a push).

    Usage:

     review-aggregate [OPTIONS] DIRECTORY

    Options:

      DIRECTORY  Directory of per-agent findings JSON files.  [required]
  --pr TEXT  PR number (default: $GITHUB_PR_NUMBER).

    review-config

    Configure the AI review backend (mutable any time).

    Usage:

     review-config [OPTIONS] COMMAND [ARGS]...

    clear

    Remove the persisted backend choice; AI review reverts to skip-neutral.

    Usage:

     review-config clear [OPTIONS]

    set-backend

    Persist an AI review backend choice for this project.

    Without --yes, prints the cost caveat and prompts for confirmation.

    Usage:

     review-config set-backend [OPTIONS] BACKEND

    Options:

      BACKEND  Backend to enable: 'api' or 'subagent'.  [required]
  --yes    Skip the confirmation prompt.

    show

    Show the currently persisted AI review backend choice.

    Usage:

     review-config show [OPTIONS]

    template-map

    Annotate rendered-project findings with best-guess template-source paths.

    Usage:

     template-map [OPTIONS]

    Options:

      --findings TEXT       Path to the findings/ dir (per-agent JSON).
                        [required]
  --template-root TEXT  Path to src/framework_cli/template.  [required]
  --package-name TEXT   package_name used in the render.  [default: demo]
  --out TEXT            Output markdown path (default: <findings>/../path-
                        map.md).

    template-render

    Render the bundled template into OUT (deterministic, non-interactive).

    Uses the canonical fixture answers (package_name=demo) plus the chosen batteries (default: all), then git-inits + commits so review tooling sees a clean repo. Produces the audit subject for a template audit (render → framework audit --target project → framework template-map).

    Usage:

     template-render [OPTIONS]

    Options:

      --out TEXT        Target directory to render into.  [required]
  --batteries TEXT  'all' (default) or a comma-separated battery subset.
                    [default: all]

    upgrade

    Move a project onto a newer framework release, then run its tests.

    Usage:

     upgrade [OPTIONS] NAME

    Options:

      NAME       Path to the project to upgrade.  [required]
  --to TEXT  Target release tag (default: the latest release).

    upskill

    Update a project to a newer framework version, then run its tests.

    Usage:

     upskill [OPTIONS] NAME

    Options:

      NAME           Path to the project to upskill.  [required]
  --with TEXT    Add a battery to the project (repeatable).
  --alerts TEXT  Reconfigure alert channels (comma-separated; replaces the
                 set).
