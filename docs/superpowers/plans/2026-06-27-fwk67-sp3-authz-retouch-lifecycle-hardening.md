# FWK67 (SP3) — multitenantauth authz re-touch + tenant-lifecycle routes — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the three control-plane tenant-lifecycle routes (deactivate/suspend/reactivate, resource-grant, rename-slug) with their preconditions met, close the DV-5 t1–t4 resolver residuals, and fold in the SP2 review carry-overs — completing the multitenantauth de-fork Phase 2.

**Architecture:** Template-payload changes to the `multitenantauth` battery. Nearly the whole code surface is integrity-LOCKED (`routes/*`, `authz/service.py`, `tenancy/registry.py`, `db/control/*`, `deps.py`), so this is one heavy all-Opus review branch — order front-loads the control-plane data model so checksums/interfaces settle, then the routes, then the (mostly light-review) policy-catalog + fitness + hardening work. All routes are control-plane (`control_session`); no per-tenant engine session ships, so SP1's data-plane guards stay deferred.

**Tech Stack:** Python 3.12, FastAPI, SQLAlchemy 2.0 (`Mapped`/`mapped_column`), Alembic (the `migrations_control/` chain), pytest, real Postgres for control-plane tests, Copier templating, `uv`.

**Spec:** `docs/superpowers/specs/2026-06-27-fwk67-sp3-authz-retouch-lifecycle-hardening-design.md`

## Global Constraints

- **Template payload, not framework source.** Every `Create`/`Modify` path under `src/framework_cli/template/` is rendered into generated projects. Do NOT lint/type-check it as framework code. Validate by **rendering + testing in the generated project** ([[template-payload-tdd-loop]]).
- **The build loop for every code task** (because template tests run in the rendered project, whose venv has deps the framework venv lacks):
  1. Render once: `TMPDIR=/var/tmp uv run framework new --batteries multitenantauth demo` into a scratch dir (e.g. `$WORK=/var/tmp/fwk67`), `cd $WORK/demo`, `uv sync`.
  2. Bring up control-plane Postgres for tests (the rendered project ships the compose + the battery-gated conftest fixtures `ctrl_engine`/`control_db_url`/`drop_tenant_db`/`truncate_control` from FWK61 Task 0).
  3. Edit the **source** file under `src/framework_cli/template/...`, then **mirror** it into `$WORK/demo`: copy `.py` verbatim; for `.jinja`, render+copy. (`.copier-answers.yml` package name is `demo` → the canonical render.)
  4. Run the target test inside `$WORK/demo` with `uv run pytest ...`.
  5. After edits: `uv run ruff format --check .` against the RENDERED output ([[ruff-format-check-after-inline-edits]], [[release-readiness-needs-render-not-local-gate]]).
- **Locked files:** editing an already-locked template file needs no manifest change (the rendered `.framework/integrity.lock` is regenerated by `framework new`). Adding a **new** locked file (a migration) requires a `BATTERY_LOCKED_SRC` entry in `src/framework_cli/integrity/classes.py`; `tests/integrity/test_auth_mechanism_lock.py` fails if a mechanism file is missing.
- **Control-plane only.** No route opens a per-tenant engine session. Do NOT wire `tenant_db`/`active_tenant`. The SP1 data-plane guards remain deferred.
- **TDD is required:** failing test first → confirm red → minimal impl → confirm green → commit.
- **Audit on real change only:** mirror the existing services — an idempotent no-op records no audit row.
- **Generic error details:** never echo a raw registry/service message that can name another tenant's opaque id (mirror `routes/tenants.py`'s `_GENERIC_*` constants).
- **Migration ids:** the control chain is `c0001`→`c0002`→`c0003`; new revisions are `c0004` (down_revision `c0003`) then `c0005` (down_revision `c0004`).

> **Path shorthand in this plan:** `MTA/` = `src/framework_cli/template/src/{{package_name}}/{% if "multitenantauth" in batteries %}multitenantauth{% endif %}/` · `CTL/` = `src/framework_cli/template/src/{{package_name}}/db/{% if "multitenantauth" in batteries %}control{% endif %}/` · `MIG/` = `src/framework_cli/template/migrations_control/versions/` · `TST/` = `src/framework_cli/template/tests/`. class TenantLifecycleEvent(ControlBase):
```python
    """Append-only audit of tenant-lifecycle mutations (suspend/reactivate/rename).
    `actor_id` NULL ⇒ a system/operator-tooling action. `detail` carries 'old_slug→new_slug' for rename."""

    __tablename__ = "tenant_lifecycle_event"

    id: Mapped[uuid.UUID] = mapped_column(Uuid, primary_key=True, default=uuid.uuid4)
    actor_id: Mapped[uuid.UUID | None] = mapped_column(Uuid, ForeignKey("app_user.id"))
    tenant_id: Mapped[str] = mapped_column(String(64), ForeignKey("tenant.id", ondelete="CASCADE"), nullable=False)
    action: Mapped[str] = mapped_column(String(16), nullable=False)
    detail: Mapped[str | None] = mapped_column(Text)
    at: Mapped[datetime] = mapped_column(DateTime(timezone=True), server_default=func.now(), nullable=False)

    __table_args__ = (
        CheckConstraint("action IN ('suspend', 'reactivate', 'rename')", name="ck_tle_action"),
        Index("ix_tle_tenant_at", "tenant_id", "at"),
    )
```

Ensure `Uuid`, `ForeignKey`, `Text`, `CheckConstraint`, `Index`, `func` are imported in `tenant.py` (add any missing). Export `TenantLifecycleEvent` from `CTL/models/__init__.py`.

```python
def reactivate_tenant(s: Session, tenant_id: str) -> None:
    """Operator reactivation: suspended → active. 404 if absent; 409 if not currently suspended."""
    tenant = s.get(m.Tenant, tenant_id)
    if tenant is None:
```

```python
_PARAM = re.compile(r"\{(\w+)\}")


def _resource_leaf_params(on: str) -> set[str]:
    """Params referenced in the resource segment of a Perm `on` pattern (substring after '/resource:')."""
    seg = on.split("/resource:", 1)[1] if "/resource:" in on else ""
    return set(_PARAM.findall(seg))


def test_DV5_t2_resource_leaves_bind_canonical_resource_id(app: FastAPI) -> None:
    """PER-LEAF: every Perm leaf with '/resource:' in `on` must bind exactly {resource_id}.
    Route-level resource_params() is insufficient — it passes a multi-resource over-grant."""
    offenders = []
    for route in _api_routes(app):
        auth = _authorized(route)
        if auth is None:
            continue
        for leaf in auth.perm_leaves():
            if "/resource:" in getattr(leaf, "on", ""):
                if _resource_leaf_params(leaf.on) != {"resource_id"}:
                    offenders.append(f"{route.path} :: {leaf.on}")
    assert offenders == [], f"resource leaves not bound to {{resource_id}}: {offenders}"


def test_DV5_t2_route_level_check_is_insufficient() -> None:
```
