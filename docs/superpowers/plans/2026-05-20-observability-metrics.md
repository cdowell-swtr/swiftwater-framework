# Observability Metrics Core (Plan 3b-1) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a local metrics-observability stack to generated projects — Prometheus scraping the app's `/metrics`, Grafana auto-provisioning a datasource + an SLO dashboard, and Alertmanager with alert rules — where the **Grafana dashboard and Prometheus alert rules are generated from the SLO definitions** established in Plan 3a (single source of truth).

**Architecture:** A pure generator (`observability/provisioning.py`) turns the `SLO` list from `observability/slo.py` into a Prometheus rule-group dict and a Grafana dashboard dict — fully unit-testable, no I/O. A thin script (`scripts/gen_observability.py`) serialises those into `infra/observability/` (committed so `task dev` works), regenerated by `task observability:gen`; a drift test asserts the committed files match the generator. Prometheus/Grafana/Alertmanager run as Compose services in the **`dev` profile only** (not `lite`), scraping `app:8000/metrics`. This is **Plan 3b-1**; logs (Loki/Promtail → 3b-2) and traces (Tempo/OTEL → 3b-3) are separate plans.

**Tech Stack:** Prometheus, Grafana (file provisioning), Alertmanager; Python (pure generator + `PyYAML` for serialisation); Docker Compose profiles; Copier/Jinja; `uv`; `pytest` (+ `PyYAML` already in the framework's dev deps from 3a Task 7).

**Spec reference:** `docs/superpowers/specs/2026-05-20-framework-design.md` — §8 (Observability Stack: Prometheus scrapes `/metrics`; Grafana dashboards + Alertmanager rules auto-generated from SLO definitions; alerts on SLO breach) and §3 (`infra/observability/` tree). Builds on Plan 3a (merged to `master`): `observability/slo.py` (`SLO`, `default_slos`), `observability/metrics.py` (exposes `app_requests_total`, `app_request_errors_total`, `app_request_latency_p99_ms`, `app_up`), and the `dev`/`lite`/`test` Compose profiles in `infra/compose/`.

**Scope boundaries (NOT in this plan):**
- **Logs** (Loki + Promtail, ship structlog JSON) → **Plan 3b-2**.
- **Traces** (Tempo + OpenTelemetry instrumentation + the unified OTEL Collector pipeline) → **Plan 3b-3**. No OTEL collector here; Prometheus scrapes the app directly.
- **Alert routing to real channels** (Slack/email/PagerDuty from the `framework new` wizard) → deferred (spec §3 wizard). Alertmanager here uses a local null/default receiver; alerts are visible in the Alertmanager UI.
- **Grafana auth/SMTP/persistence**: dev defaults (anonymous admin, no persistence) — fine for local.

## Design Decisions (made per "decide & document")

1. **SLO → PromQL mapping lives in `provisioning.py`** (not on the `SLO` model), keyed by `SLO.key`, because the mapping is observability-specific knowledge and the metric names belong to `metrics.py`. `request_latency_p99_ms` → the `app_request_latency_p99_ms` gauge directly; `error_rate_pct` → `100 * app_request_errors_total / clamp_min(app_requests_total, 1)` (computed in PromQL from the two counters, since `metrics.py` exposes counters, not a rate gauge). A guard test asserts every default SLO key has a mapping, so adding an SLO without a mapping fails loudly.
2. **Generated output is committed** (`slo_alerts.yml`, `slo.json`) so a freshly scaffolded project's `task dev` has them on day one; `task observability:gen` regenerates after an SLO change, and a **drift test** (in the generated project's suite) asserts the committed files equal the generator's output for the default SLOs — like an OpenAPI-drift check.
3. **Observability runs in the `dev` profile only** — `lite` stays app-only (resource-light), consistent with 3a. `task dev` brings up app + Traefik + Prometheus + Grafana + Alertmanager.
4. **The generated project gains a `pyyaml` dev dep** (the gen script + drift test serialise/parse YAML). It's dev-only.
5. **Grafana provisioning layout:** `infra/observability/grafana/provisioning/{datasources,dashboards}/*.yml` (mounted to `/etc/grafana/provisioning`) + `infra/observability/grafana/dashboards/slo.json` (mounted to `/var/lib/grafana/dashboards`, where the dashboard provider points). The datasource `uid: prometheus` matches the dashboard panels' `datasource.uid`.
6. **No new Copier questions** (YAGNI).

```python
"""Generate Prometheus alert rules and a Grafana dashboard from the SLO definitions.

Pure functions: SLOs in, config dicts out. `observability/slo.py` is the single source of
truth; `scripts/gen_observability.py` serialises these into `infra/observability/`.

Each SLO key maps to the PromQL expression for its *current value*, over the metrics
exposed by `observability/metrics.py` (`/metrics`). Adding a new SLO requires adding its
PromQL here — `test_provisioning.py` asserts every default SLO key is mapped.
"""

from __future__ import annotations

from typing import TYPE_CHECKING

if TYPE_CHECKING:
    from .slo import SLO

# PromQL for each SLO's current value. error_rate_pct is computed from the two counters
# (metrics.py exposes counters, not a rate gauge); clamp_min avoids divide-by-zero.
SLO_PROMQL: dict[str, str] = {
    "request_latency_p99_ms": "app_request_latency_p99_ms",
    "error_rate_pct": "100 * app_request_errors_total / clamp_min(app_requests_total, 1)",
}


def _alert_name(key: str) -> str:
    return "".join(part.capitalize() for part in key.split("_")) + "Breached"


def prometheus_alert_rules(slos: list[SLO]) -> dict:
    """A Prometheus rule-group dict (serialise to YAML under rule_files)."""
    rules = [
        {
            "alert": _alert_name(slo.key),
            "expr": f"{SLO_PROMQL[slo.key]} > {slo.threshold}",
            "for": "1m",
            "labels": {"severity": "warning", "slo": slo.key},
            "annotations": {
                "summary": f"{slo.description} above SLO ({slo.threshold}{slo.unit})"
            },
        }
        for slo in slos
    ]
    return {"groups": [{"name": "slo", "rules": rules}]}
```

```python
def grafana_dashboard(slos: list[SLO]) -> dict:
    """A minimal Grafana dashboard dict: one timeseries panel per SLO, value vs threshold."""
    panels = [
        {
            "id": i + 1,
            "title": f"{slo.description} (SLO {slo.threshold}{slo.unit})",
            "type": "timeseries",
            "datasource": {"type": "prometheus", "uid": "prometheus"},
            "gridPos": {"h": 8, "w": 12, "x": (i % 2) * 12, "y": (i // 2) * 8},
            "targets": [{"refId": "A", "expr": SLO_PROMQL[slo.key]}],
            "fieldConfig": {
                "defaults": {
                    "unit": slo.unit,  # "ms"/"percent" are valid Grafana unit ids
                    # thresholdsStyle is what actually DRAWS the threshold line on a timeseries
                    "custom": {"thresholdsStyle": {"mode": "line"}},
                    "thresholds": {
                        "mode": "absolute",
                        "steps": [
                            {"color": "green", "value": None},
                            {"color": "red", "value": slo.threshold},
                        ],
                    },
                },
                "overrides": [],
            },
        }
        for i, slo in enumerate(slos)
    ]
    return {
        "uid": "slo",
        "title": "SLOs",
        "tags": ["slo"],
        "schemaVersion": 39,
        "version": 1,
        "time": {"from": "now-1h", "to": "now"},
        "panels": panels,
    }
```

```python
"""Regenerate infra/observability config from the SLO definitions. Run via `task observability:gen`.

Pure transforms live in observability/provisioning.py; this only serialises + writes files.
"""

from __future__ import annotations

import json
from pathlib import Path

import yaml

from {{ package_name }}.config.settings import get_settings
from {{ package_name }}.observability.provisioning import (
    grafana_dashboard,
    prometheus_alert_rules,
)
from {{ package_name }}.observability.slo import default_slos

ROOT = Path(__file__).resolve().parent.parent


def main() -> None:
    slos = default_slos(get_settings())

    alerts = ROOT / "infra/observability/prometheus/alerts/slo_alerts.yml"
    alerts.parent.mkdir(parents=True, exist_ok=True)
    alerts.write_text(yaml.safe_dump(prometheus_alert_rules(slos), sort_keys=False))

    dashboard = ROOT / "infra/observability/grafana/dashboards/slo.json"
    dashboard.parent.mkdir(parents=True, exist_ok=True)
    dashboard.write_text(json.dumps(grafana_dashboard(slos), indent=2) + "\n")


if __name__ == "__main__":
    main()
```

```python
@pytest.mark.skipif(not _docker_available(), reason="uv and docker are required for the live-stack test")
def test_rendered_project_dev_stack_prometheus_scrapes_app(tmp_path: Path):
    dest = tmp_path / "demo"
    render_project(dest, DATA)
    assert subprocess.run(["uv", "lock"], cwd=dest).returncode == 0
    # certs are required by the dev profile's traefik precondition path, but compose
    # itself doesn't need them for prometheus/app; bring up the dev profile directly.
    base, dev = "infra/compose/base.yml", "infra/compose/dev.yml"
    up = ["docker", "compose", "-f", base, "-f", dev, "--profile", "dev", "up", "-d", "--build"]
    down = ["docker", "compose", "-f", base, "-f", dev, "--profile", "dev", "down", "-v"]
    assert subprocess.run(up, cwd=dest).returncode == 0
    try:
        deadline = time.time() + 120
        up_targets = None
        while time.time() < deadline:
            try:
                with urllib.request.urlopen(
                    "http://localhost:9090/api/v1/targets?state=active", timeout=3
                ) as resp:
                    data = json.loads(resp.read())
                    actives = data.get("data", {}).get("activeTargets", [])
                    app_targets = [t for t in actives if t.get("labels", {}).get("job") == "app"]
                    if app_targets and app_targets[0].get("health") == "up":
                        up_targets = app_targets
                        break
            except OSError:
                pass
            time.sleep(3)
        assert up_targets, "prometheus did not report the app target healthy within 120s"
    finally:
        subprocess.run(down, cwd=dest)
```
