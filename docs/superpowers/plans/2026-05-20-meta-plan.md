## Proposed Sub-Slicing for Plan 3 (largest item)

Plan 3 bundles several subsystems; when written it should be split so each is buildable and testable on its own:

- **3a ‚ÄĒ Local runtime:** Docker Compose `base`/`dev`/`dev:lite`/`test` profiles, Traefik + mkcert local HTTPS, hot reload, container startup ordering (healthchecks + `depends_on: service_healthy`), `.gitattributes`/TZ=UTC consistency, `SERVICES.md`. Plus structlog + correlation IDs and a *real* SLO-evaluating `/health` (replacing the walking-skeleton stub). Testable: `task dev` brings up the app over HTTPS; `/health` returns live SLO status.
- **3b ‚ÄĒ Observability stack:** Prometheus, Grafana (auto-provisioned dashboards from SLO defs), Loki + Promtail, Tempo (traces), Alertmanager, OTEL Collector ‚ÄĒ wired so a dev sees their dashboard locally. Alert channels come from the `framework new` wizard.
- **3c ‚ÄĒ Database lifecycle:** the `database` paradigm wizard's relational path (PostgreSQL + SQLAlchemy + Alembic), DB init/migrate/seed on first `task dev`, isolated test DB in the `test` profile. (Other paradigms / the full wizard may fold into Plan 8's `database` battery.)

This split is a proposal to confirm at the start of Plan 3, not a commitment.
