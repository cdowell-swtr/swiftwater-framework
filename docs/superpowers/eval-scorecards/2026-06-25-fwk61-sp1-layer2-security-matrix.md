## Confirmed findings to fix now

1. **P1 — I-CRED credential disclosure (High, in-battery / integrity-locked).** `migrate_tenant` (`src/demo/multitenantauth/tenancy/provision.py:56`) does `cfg.set_main_option("sqlalchemy.url", dsn)` on the raw tenant DSN; a `%` anywhere in the DSN (BYO literal `%`, OR the default-derived path where `dsn.py:21` `render_as_string(hide_password=False)` percent-encodes any special char in the app DB password) makes stdlib `ConfigParser.BasicInterpolation.before_set` raise `ValueError("invalid interpolation syntax in %r …")` whose `%r` embeds the **full plaintext DSN incl. password**. Unhandled in `migrate_tenant`/`provision_tenant` (provision.py:52-57,83-88) → propagates to caller logs/CLI stderr/tracebacks. **Fix (must ship in the locked battery file):** `cfg.set_main_option("sqlalchemy.url", dsn.replace("%", "%%"))` — `get`-time `BasicInterpolation` un-escapes `%%`→`%`, and `migrations/env.py` reads via `config.get_main_option`/`config.get_section`, so the DSN round-trips intact; or pass the URL out-of-band via `cfg.attributes["tenant_url"]` (attributes are not interpolated). Add a regression test: provision with a `%`-bearing password → no `ValueError`, no DSN substring in any captured log/message.
2. **P10 — I-IDEMP control-plane DROP (High).** `src/demo/multitenantauth/.../migrations/env.py:6-15` imports `ControlBase` but never `demo.db.control.models`, so `_CONTROL_TABLES` (env.py:15) is empty and `_include_object` (env.py:20-23) excludes no control table. **Fix:** add `from demo.db.control import models  # noqa: F401  (registers control tables on ControlBase.metadata)` before line 15 (mirroring `migrations_control/env.py:7`); add a regression test asserting `set(ControlBase.metadata.tables)` under the app env import set is non-empty and contains the control table names.
3. **P3 — I-BUDGET fail-open knob (Medium).** `settings.py:94` `db_pool_safety_factor: float = 0.8` → `db_pool_safety_factor: float = Field(default=0.8, gt=0.0, le=1.0)`. A `>1` value (or a `80`-for-`0.8` typo) can no longer inflate the budget past the live ceiling. *Degrade-to-permissive config knob — FIX-NOW even though the Crit/High gate is driven by P1/P10.*
4. **P4 — I-BUDGET fail-open knobs (Medium).** `settings.py:88-92`: add floors mirroring `max_cached_engines=Field(ge=1)` — `tenant_pool_size`/`control_pool_size` → `Field(ge=1)` (forbid the `0`=unbounded sentinel); `tenant_max_overflow`/`control_max_overflow` → `Field(ge=0)` (forbid the `-1`=unlimited sentinel). This makes `required_connections` a true upper bound. *Degrade-to-permissive config knob — FIX-NOW.*

**Maintainer DECISION (P5 — I-BUDGET accounting, Medium, fail-open at defaults-divergence):** the control-plane budget term (`engine_registry.py:32-36`) is computed from `control_pool_size`/`control_max_overflow` settings, but the real control engine (`db/control/engine.py:21` → `db/engine.py:9-11`) ignores them (fixed default 5+10=15). Correct at defaults; under-states `required` if an operator lowers the control knobs. Two valid fixes — (a) thread the knobs through `build_engine` into the control engine (preferred — makes the knobs real and the term truthful both directions), or (b) remove the phantom settings and derive the control term from the engine's actual pool config. This is a degrade-to-permissive knob; flagged as DECISION only because the fix shape is a maintainer call, not because it can wait.

## Phase-2 preconditions for Meridian (record, do not fix in SP1)

1. **Data-plane sentinel guard (from P7).** When SP2 wires `tenant_db` to a route, add a fail-closed guard in `session._resolve_dsn`/`tenant_db` refusing to route any tenant whose resolved DSN is the un-provisioned placeholder (host `unprovisioned` / `== _PLACEHOLDER_DSN`) → LookupError→404/409. Makes isolation structural rather than resting on the non-routable hostname.
2. **Parse-before-cache (from P8).** Before SP2 route wiring, validate `make_url(dsn)` in `_resolve_dsn` before the cache write and map engine-build/`ArgumentError` on the pre-yield path to a fail-closed 404, so a malformed-but-truthy DSN can never poison `_dsn_cache` for the TTL.
3. **Lock-hygiene + `connect_timeout` (from P6 + `_evict` disposition).** Build/budget-probe the engine outside `self._lock`; set `connect_args={"connect_timeout": N}`; negative-cache a failed endpoint; dispose evicted pools outside the lock. Lands when `tenant_db` first reaches a route.
4. **`/metrics` topology posture (from P11).** Keep `endpoint_of` host:port-only (never widen labels to dbname); once cross-cluster real DSNs are possible, auth-/network-scope `/metrics`.
5. **DDL sink hardening (from P2).** Before any future code wires `create_database` to a route or BYO-DSN input: validate `dbname` against `^[a-z0-9_]+$` at the sink and add a `tenant_db_name_prefix` field_validator.
6. **Resolver-seam credential contract (from P12).** When the secrets-backed resolver lands, normalize the `LookupError` branch (`session.py:89-90`) to `raise LookupError(f"DSN resolution failed for {tenant_id!r}") from None`, and test-enforce the "resolver LookupError must be DSN-free" contract.
